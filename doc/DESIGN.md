# torc — C++ Package Manager

**torc** (Old Celtic: twisted metal ring; cf. English "torque") — a minimal,
Make-native C++ dependency manager.

## Philosophy

- Flat by default, plugins for transitive discovery
- Make is the build system — torc generates includable `.mak` fragments
- Shared dependency directory (user/group level, setgid for collaboration)
- Explicit over clever: checksums, versioned dirs, no hidden magic
- Parallel fetching and building
- Clean housekeeping of stale versions

## Architecture

```
torc install      Fetch, verify, build, install deps → depdir
torc clean        Remove stale/unused versions from depdir
torc generate     Emit extdep.mak + localdep.mak
torc update       Check for newer versions, update manifest
torc list         Show installed packages and status
torc help         Auto-generated from option declarations
```

## Dependency Directory Layout

```
~/.local/share/torc/           (default, XDG_DATA_HOME)
  fmt/10.1.1/
    include/
    lib/
  spdlog/1.12.0/
    include/
    lib/
```

- User-level: `~/.local/share/torc/` (default)
- Group-level: `/opt/torc/` with `g+S` (setgid) for shared team use
- Project-local: `./deps/` (override via manifest or CLI)

## Manifest Format (mini-YAML)

```yaml
# torc.yaml
depdir: ~/.local/share/torc
parallel: 4

packages:
  - name: fmt
    version: 10.1.1
    source: https://github.com/fmtlib/fmt/archive/refs/tags/10.1.1.tar.gz
    sha256: abc123def456...
    build: |
      cmake -B build -DCMAKE_INSTALL_PREFIX=${PREFIX} -DFMT_TEST=OFF
      cmake --build build -j${JOBS}
      cmake --install build

  - name: spdlog
    version: 1.12.0
    source: https://github.com/gabime/spdlog/archive/v1.12.0.tar.gz
    sha256: 789abc...
    build: |
      cmake -B build -DCMAKE_INSTALL_PREFIX=${PREFIX} -DSPDLOG_FMT_EXTERNAL=ON
      cmake --build build -j${JOBS}
      cmake --install build
    discover: spdlog.discover
```

## Generated Output

`extdep.mak`:
```makefile
# Auto-generated by torc — do not edit
TORC_CXXFLAGS = -I$(HOME)/.local/share/torc/fmt/10.1.1/include \
                -I$(HOME)/.local/share/torc/spdlog/1.12.0/include
TORC_LDFLAGS  = -L$(HOME)/.local/share/torc/fmt/10.1.1/lib \
                -L$(HOME)/.local/share/torc/spdlog/1.12.0/lib
TORC_LIBS     = -lfmt -lspdlog
```

## Plugin System (Transitive Discovery)

A `.discover` file is an executable (script or binary) that:
- Receives: package name, version, install prefix on stdin/args
- Outputs: additional dependency entries in the same YAML format to stdout
- Core merges results, detects version conflicts, reports errors

## Exit Codes

All exit codes follow `<sysexits.h>`:
- 0: EX_OK
- 64: EX_USAGE (bad CLI usage)
- 65: EX_DATAERR (manifest parse error)
- 66: EX_NOINPUT (file not found)
- 69: EX_UNAVAILABLE (network/download failure)
- 73: EX_CANTCREAT (cannot create depdir)
- 74: EX_IOERR (I/O error during build)
- 78: EX_CONFIG (bad configuration)

## TUI Elements (Optional)

- Progress bars for downloads (when stderr is a terminal)
- Colored status output (respects NO_COLOR)

## Scaffold New Project (`torc new`)

Bootstrap a project from scratch:

```bash
torc new myproject [--lib] [--no-git]
```

Creates:
```
myproject/
├── torc.yaml          # empty packages list, depdir set
├── Makefile           # generated (same as torc init)
├── src/
│   └── main.cpp      # minimal hello-world (or lib.hpp+lib.cpp with --lib)
├── include/
│   └── myproject/     # public headers (empty, or lib.hpp with --lib)
├── tests/
│   └── test_main.cpp # minimal test
├── .gitignore         # build/, *.o, *.d, extdep.mak
└── .clang-format      # project defaults
```

With `--lib`: generates a library skeleton (no `main()`, exports a header).

Unless `--no-git`: runs `git init` + sets `core.hooksPath` if `hooks/` exists.

## Direct Build (`torc build`)

For simple project layouts, torc can compile and link without a Makefile:

```bash
torc build [--src=src] [--out=build] [--target=myapp] [--std=c++20] [--release]
```

Behavior:
- Discovers `*.cpp` under `--src` (default `src/`)
- Compiles each to `--out/*.o` with `-MMD -MP` for incremental rebuilds
- Links into `--out/target`
- Includes `TORC_CXXFLAGS`/`TORC_LDFLAGS`/`TORC_LIBS` from installed manifest
- Respects `.d` files — only recompiles changed sources (like Make)
- `--release`: `-O2 -DNDEBUG`; default: `-O0 -g`
- Parallel compilation (uses `-j` / `parallel:` from manifest)

This is intentionally limited to flat/simple layouts:
- Single `src/` directory (no recursive subdirs unless `--recursive`)
- One binary target
- No library outputs, no custom per-file flags

For anything more complex: use `torc init` to generate a proper Makefile.

The point is zero-friction for small projects and prototypes — no Makefile needed
to get started, just `torc.yaml` + a `src/` directory.

```bash
# Minimal workflow — no Makefile at all:
$ ls
src/main.cpp  src/util.cpp  src/util.hpp  torc.yaml

$ torc install    # fetch deps
$ torc build      # compile + link → build/myapp
$ ./build/myapp
```

## Project Makefile Generation (`torc init`)

Generate a starter Makefile for a new C++ project in the specified directory (default `./`):

```bash
torc init [--dir=path] [--name=myproject] [--force]
```

Safety:
- If `Makefile` already exists: **fail with EX_CANTCREAT** and a diagnostic
- With `--force`: rename existing to `Makefile.YYYYMMDD-HHMMSS.bak`, then generate
- Never silently overwrite

Produces:

```makefile
# Generated by torc — customize freely
CXX       ?= g++
CXXSTD    ?= -std=c++20
WARNINGS  ?= -Wall -Wextra -Werror -pedantic
CXXFLAGS  += $(CXXSTD) $(WARNINGS) -Isrc

# ── torc dependencies ───────────────────────────────────
-include extdep.mak
CXXFLAGS += $(TORC_CXXFLAGS)
LDFLAGS  += $(TORC_LDFLAGS)
LDLIBS   += $(TORC_LIBS)

# ── Local header dependencies (auto-generated by -MMD) ──
-include $(OBJS:.o=.d)

SRC_DIR   = src
BUILD_DIR = build
SRCS      = $(wildcard $(SRC_DIR)/*.cpp)
OBJS      = $(patsubst $(SRC_DIR)/%.cpp,$(BUILD_DIR)/%.o,$(SRCS))
TARGET    = $(BUILD_DIR)/myproject

DEPFLAGS  = -MMD -MP

.PHONY: all clean deps

all: $(TARGET)

$(TARGET): $(OBJS) | $(BUILD_DIR)
	$(CXX) $(CXXFLAGS) $(LDFLAGS) -o $@ $^ $(LDLIBS)

$(BUILD_DIR)/%.o: $(SRC_DIR)/%.cpp | $(BUILD_DIR)
	$(CXX) $(CXXFLAGS) $(DEPFLAGS) -c -o $@ $<

$(BUILD_DIR):
	mkdir -p $@

# Fetch/build external deps and regenerate extdep.mak
deps:
	torc install
	torc generate

clean:
	rm -rf $(BUILD_DIR)
```

## Hooking Into Existing Makefiles (`torc hook`)

For projects that already have a Makefile, inject torc integration non-destructively:

```bash
torc hook [--makefile=Makefile]
```

Inserts (or updates) a clearly-marked block:

```makefile
# ── BEGIN torc ──────────────────────────────────────────
-include extdep.mak
CXXFLAGS += $(TORC_CXXFLAGS)
LDFLAGS  += $(TORC_LDFLAGS)
LDLIBS   += $(TORC_LIBS)
# ── END torc ────────────────────────────────────────────
```

Rules:
- Inserts after the first variable assignment block (before first rule)
- If `# ── BEGIN torc` already exists, replaces the block in-place
- Creates a `.bak` backup before modifying
- Exits with EX_NOINPUT if Makefile not found

## Local Header Dependencies (`-MMD`)

The generated/hooked Makefile uses `-MMD -MP` for automatic header dependency tracking:

```makefile
DEPFLAGS = -MMD -MP

$(BUILD_DIR)/%.o: $(SRC_DIR)/%.cpp | $(BUILD_DIR)
	$(CXX) $(CXXFLAGS) $(DEPFLAGS) -c -o $@ $<

-include $(OBJS:.o=.d)
```

This is **direct** — GCC/Clang emit `.d` files alongside `.o` files, Make includes them
on subsequent builds. No torc involvement at runtime; it's purely a Makefile pattern
that `torc init` sets up correctly.

For **indirect** (inter-library) dependencies within a multi-directory project,
`torc generate` can also emit `localdep.mak`:

```makefile
# localdep.mak — internal project libraries
LOCALDEP_CXXFLAGS = -Ilib/core/include -Ilib/util/include
LOCALDEP_LDFLAGS  = -Llib/core/build -Llib/util/build
LOCALDEP_LIBS     = -lcore -lutil
LOCALDEP_ORDER    = lib/util lib/core   # build order (leaves first)
```

## CLion / compile_commands.json (`torc compdb`)

Generate `compile_commands.json` for IDE integration (CLion, clangd, ccls):

```bash
torc compdb [--dir=build]
```

Scans the project source tree and emits:

```json
[
  {
    "directory": "/home/user/myproject",
    "command": "g++ -std=c++20 -Wall -Wextra -Isrc -I/home/user/.local/share/torc/fmt/10.1.1/include -I/home/user/.local/share/torc/spdlog/1.12.0/include -MMD -MP -c -o build/main.o src/main.cpp",
    "file": "src/main.cpp"
  },
  ...
]
```

Sources of information:
- `CXXFLAGS` from Makefile (parsed or passed via `--cxxflags=...`)
- `TORC_CXXFLAGS` from the installed manifest (include paths)
- Source files discovered by globbing `src/**/*.cpp` (or from manifest `sources:` key)

Alternative: `torc compdb --bear` wraps `make` with Bear-style interception
(intercepts compiler calls via `LD_PRELOAD`) for exact commands. Fallback for
complex Makefiles where static analysis isn't sufficient.

Output location: `compile_commands.json` in project root (where CLion expects it),
or `build/compile_commands.json` with `--dir=build` + symlink in root.

## Security

- SHA-256 checksum verification on all downloads (mandatory)
- No execution of anything before checksum passes
- Manifest is the single source of truth
