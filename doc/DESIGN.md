# torc — C++ Package Manager

**torc** (Old Celtic: twisted metal ring; cf. English "torque") — a minimal,
Make-native C++ dependency manager.

## Philosophy

- Flat by default, plugins for transitive discovery
- Make is the build system — torc generates includable `.mak` fragments
- Shared dependency directory (user/group level, setgid for collaboration)
- Explicit over clever: checksums, versioned dirs, no hidden magic
- Parallel fetching and building
- Clean housekeeping of stale versions

## Architecture

```
torc install      Fetch, verify, build, install deps → depdir
torc clean        Remove stale/unused versions from depdir
torc generate     Emit extdep.mak + localdep.mak
torc update       Check for newer versions, update manifest
torc list         Show installed packages and status
torc help         Auto-generated from option declarations
```

## Dependency Directory Layout

```
~/.local/share/torc/           (default, XDG_DATA_HOME)
  fmt/10.1.1/
    include/
    lib/
  spdlog/1.12.0/
    include/
    lib/
```

- User-level: `~/.local/share/torc/` (default)
- Group-level: `/opt/torc/` with `g+S` (setgid) for shared team use
- Project-local: `./deps/` (override via manifest or CLI)

## Manifest Format (mini-YAML)

```yaml
# torc.yaml
depdir: ~/.local/share/torc
parallel: 4

packages:
  - name: fmt
    version: 10.1.1
    source: https://github.com/fmtlib/fmt/archive/refs/tags/10.1.1.tar.gz
    sha256: abc123def456...
    build: |
      cmake -B build -DCMAKE_INSTALL_PREFIX=${PREFIX} -DFMT_TEST=OFF
      cmake --build build -j${JOBS}
      cmake --install build

  - name: spdlog
    version: 1.12.0
    source: https://github.com/gabime/spdlog/archive/v1.12.0.tar.gz
    sha256: 789abc...
    build: |
      cmake -B build -DCMAKE_INSTALL_PREFIX=${PREFIX} -DSPDLOG_FMT_EXTERNAL=ON
      cmake --build build -j${JOBS}
      cmake --install build
    discover: spdlog.discover
```

## Generated Output

`extdep.mak`:
```makefile
# Auto-generated by torc — do not edit
TORC_CXXFLAGS = -I$(HOME)/.local/share/torc/fmt/10.1.1/include \
                -I$(HOME)/.local/share/torc/spdlog/1.12.0/include
TORC_LDFLAGS  = -L$(HOME)/.local/share/torc/fmt/10.1.1/lib \
                -L$(HOME)/.local/share/torc/spdlog/1.12.0/lib
TORC_LIBS     = -lfmt -lspdlog
```

## Plugin System (Transitive Discovery)

A `.discover` file is an executable (script or binary) that:
- Receives: package name, version, install prefix on stdin/args
- Outputs: additional dependency entries in the same YAML format to stdout
- Core merges results, detects version conflicts, reports errors

## Exit Codes

All exit codes follow `<sysexits.h>`:
- 0: EX_OK
- 64: EX_USAGE (bad CLI usage)
- 65: EX_DATAERR (manifest parse error)
- 66: EX_NOINPUT (file not found)
- 69: EX_UNAVAILABLE (network/download failure)
- 73: EX_CANTCREAT (cannot create depdir)
- 74: EX_IOERR (I/O error during build)
- 78: EX_CONFIG (bad configuration)

## TUI Elements (Optional)

- Progress bars for downloads (when stderr is a terminal)
- Colored status output (respects NO_COLOR)
- Parallel build status display

## Security

- SHA-256 checksum verification on all downloads (mandatory)
- No execution of anything before checksum passes
- Manifest is the single source of truth
